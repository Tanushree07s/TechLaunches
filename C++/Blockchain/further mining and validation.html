• A cryptographic puzzle in the Proof-of-Work Blockchain is solved using hashing Process in which input data is transformed into a fixed-size, unique output hash value using a one-way function that cannot be reversed, ensuring data security and integrity. .

  • Target Hash
    • The target hash is a threshold value that defines how hard it is to mine a block in a Proof-of-Work system.
    • A block is considered valid only if its computed hash is less than or equal to this target.
    • These hashes are usually represented in hexadecimal.
    • The target determines the difficulty of the puzzle:
      • Lower target = fewer valid hashes = harder puzzle.
      • Higher target = more valid hashes = easier puzzle.
    • The difficulty of the target hash is determined by the mining difficulty

      This tells the miner how small that target is compared to the easiest it could ever be. It is adjusted using:

      • Targeted average time (desired block time) taken for generating a new block. This is decided by the blockchain protocol to ensure a consistent and stable rate of block creation.
      • Recent actual average time taken to generate new blocks.
      • For insance, in the Bitcoin system: new_difficulty = old_difficulty × (actual_time / target_time)
      .
    • Computing Target Hash:
      • current_target= max_target/difficulty
      • max_target = The largest possible target value allowed by the blockchain protocol. It corresponds to difficulty 1.
      • current_target = the actual target hash threshold right now.
      • A target with more leading zeros in hexadecimal corresponds to a smaller integer
      • Smaller number → fewer possible valid hashes → higher difficulty.

  • How hashing works in mining:
    1. Take the block data (e.g., transactions, previous hash, timestamp).
    2. Append a nonce It is a random or sequential number that miners keep changing to get a valid hash. (starting from 0).
    3. Use algorithms such as Secure Hash Algorithm 256-bit (SHA-256) It's a cryptographic hash function — a one-way mathematical function that turns input data (of any size) into a fixed-size output: a 256-bit hex string. to compute the hash of the combined data.
    4. Check if the hash has the required number of leading zeros.
    5. If yes, the block is mined.
    6. If no, increment the nonce and repeat.
